# Final Project: Clean Code & Automation Cycle

## 📌 Overview

This project focuses on identifying, analyzing, and refactoring common **code smells** while ensuring system functionality through automated testing with **Selenium**. The goal is to improve code quality, maintainability, and scalability using professional development tools such as **IntelliJ IDEA**.

---

## 🎯 Objectives

* Identify at least **10 different code smells** from multiple categories.
* Apply **refactoring techniques** to eliminate these issues.
* Ensure application stability using **automated Selenium tests**.
* Maintain a **clean Git history** with meaningful commits.
* Document all improvements in a clear and technical manner.

---

## 🧠 Code Smells Implemented

The project addresses the following categories:

### 1. Bloaters

* Long Method
* Large Class

### 2. Object-Oriented Abusers

* Switch Statements
* Temporary Fields

### 3. Change Preventers

* Divergent Change
* Shotgun Surgery

### 4. Dispensables

* Duplicate Code
* Dead Code

### 5. Couplers

* Feature Envy
* Inappropriate Intimacy

---

## 🔧 Refactoring Techniques Applied

Some of the main techniques used include:

* Extract Method
* Extract Class
* Replace Conditional with Polymorphism
* Remove Dead Code
* Reduce Code Duplication
* Encapsulation improvements

Each refactoring was applied carefully to preserve functionality while improving readability and maintainability.

---

## 🤖 Selenium Test Suite

A complete **automated testing suite** was developed to ensure:

* All features work correctly after refactoring
* No regressions are introduced
* Tests remain stable and consistently pass

---

## 🚀 Key Results

* Improved code readability and structure
* Reduced technical debt
* Increased maintainability and scalability
* Fully functional and tested system
